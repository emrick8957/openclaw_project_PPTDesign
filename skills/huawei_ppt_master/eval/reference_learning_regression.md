# 参考材料学习专项回归

## 1. 目标
验证 Skill 在持续吸收 PPT / PPT 图片后，是否真正提升了样式、结构、表达和体系层能力，同时没有引入主题污染或过拟合。

## 2. 回归项
### 2.1 视觉学习回归
检查是否学到配色、密度、标题区、页脚区、图表主形态。

失败信号：
- 出现互联网发布会风；
- 低密度口号页；
- 配色漂移；
- 页脚或标题区不稳定。

补充检查：

- 是否把“红色强调”误学成“大面积红色堆叠”；
- 是否把高密度误学成满页卡片/表格堆叠；
- 是否对卡片、表格、泳道、闭环图建立统一组件语法；
- 是否能在生成页面图片后识别“结构正确但视觉偏重”的问题。

### 2.2 结构学习回归
检查是否学到章节编排、页型分工、页面结构，而非只学到表面措辞。

失败信号：
- 只复用参考材料中的词；
- 不复用页面组织方式；
- 多页结构仍然散乱。

### 2.3 表达学习回归
检查标题是否更结论化，收口是否更像高层汇报。

失败信号：
- 标题名词化；
- 口号化；
- 没有“判断—依据—动作”链条；
- 页面没有决策或行动收口。

### 2.4 体系学习回归
检查是否能从多套材料中抽取稳定论证框架与行业套路。

失败信号：
- 仍只会学配色与版式；
- 学不会章节逻辑；
- 把单份材料的逻辑误认为通用框架。

### 2.5 主题隔离回归
检查是否因参考材料带入无关术语或错误主题框架。

失败信号：
- 非 AI 算力主题出现昇腾、NVIDIA、CUDA、GPU/NPU 等术语；
- 把参考截图中的主题内容污染到无关任务；
- 把行业专属框架套到所有主题。

### 2.6 增量稳定性回归
检查新增材料是否无依据破坏已有通用规则。

失败信号：
- 旧规则被无依据覆盖；
- 通用模板被单一样例绑架；
- 新旧规则冲突但未降级处理。

## 3. 一票否决项
- 非 AI 主题被 AI 算力术语污染；
- 截图中的具体结论被直接当成事实；
- 单客户材料写成全局规则；
- 新规则与旧规则冲突但未降级处理。

## 4. 最小测试集建议
1. 华为风格截图 20~30 张，覆盖总览、对比、架构、路线图、风险页；
2. 同行业完整 PPT 2~3 份；
3. 跨行业参考 PPT 2 份；
4. 非 AI 主题任务 2 个；
5. AI/算力主题任务 1 个。

## 5. 验收门槛
- 六类回归不得出现 P0 失败；
- 主题污染类错误一票否决；
- 至少 80% 的新输出应被 reviewer 判定为“更接近参考材料的组织方式，而非仅接近表面措辞”。

## 5.1 页面图片视觉一致性门槛

当参考材料用于生成 page_images 或 PPTX 时，必须额外满足：

- `eval/visual_scorecard.md` 总分不低于 85；
- 任一页面低于 80 必须返工；
- 正文页不得出现厚重红色压底条抢主体；
- 页面必须保留稳定标题区、主体区、页脚区；
- 视觉问题必须能追溯到对应资产：`visual_rules.md`、`layout_library.md`、`chart_patterns.md`、`generate_page_design.md` 或 Builder 实现。

## 6. 0517 多行业解决方案图片专项回归

适用范围：当复用 `tmp/PPTTemplate/0517` 沉淀资产时执行。

### 6.1 必检项

1. 是否只复用了白底、红色强调、模块化高密度、页脚一致等视觉语法，而未复制具体页面内容；
2. 是否正确区分 `multi_module_solution_overview`、`left_logic_right_proof`、`hub_spoke_ecosystem_canvas`、`metric_case_showcase_grid` 等版式；
3. 是否把“战略总览—能力拆解—场景落地—生态证明—路径收口”用于解决方案/生态展示类任务，而非所有任务；
4. 是否把具体行业词、产品名、客户名、项目名、伙伴 Logo、成果数字降级为一次性参考或待补材料；
5. 是否在内部决策材料中补充资源、风险、责任、验收，而不是停留在宣传式案例展示。

### 6.2 失败信号

- 无证据生成具体数字或案例；
- 把多行业展示结构套到项目复盘、资源申请、风险决策等不匹配任务；
- 每页都使用生态圈/案例墙，导致机械套模板；
- 标题只有“赋能/打造/领先”等口号，没有对象、抓手和动作；
- 参考材料中的行业主题污染到无关主题。
