# 章节逻辑模式

## 1. 目标
沉淀不同汇报类型中常见的章节组织方式，帮助生成更稳定的高层汇报结构。

## 2. 常见章节主线
### 2.1 战略判断型
判断 → 外部变化 → 内部差距 → 目标路径 → 资源保障 → 决策事项

### 2.2 技术方案型
现状 → 问题 → 总体架构 → 核心能力 → 实施路径 → 运维保障

### 2.3 竞品对比型
评价框架 → 竞争格局 → 分维度对比 → 差距根因 → 优先策略 → 拍板建议

### 2.4 项目复盘型
目标 → 进展 → 成果 → 问题 → 根因 → 改进闭环

### 2.5 客户方案型
客户痛点 → 方案主张 → 能力架构 → 价值证明 → 交付路径 → 合作机制

### 2.6 资源申请型
现状约束 → 风险影响 → 资源缺口 → 投入产出 → 组织机制 → 决策事项

## 3. 章节递进规则
- 前半部分回答“为什么必须做”；
- 中间部分回答“做什么、怎么做”；
- 后半部分回答“谁负责、怎么验收、需要高层拍什么板”。

## 4. 拆页与合页规则
- 一个页面只能服务一个核心判断；
- 当同页同时包含现状、问题、方案和决策时，优先拆页；
- 当两个页面只是在重复同一判断时，优先合并。
