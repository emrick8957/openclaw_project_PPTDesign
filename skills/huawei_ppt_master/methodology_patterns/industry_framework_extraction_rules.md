# 行业框架抽取规则

## 1. 目标
从持续投喂的行业 PPT、PPT 图片和材料中抽取可复用行业方法论，同时避免把单客户、单项目、单场景结论误写成全局规则。

## 2. 允许抽取的对象
- 行业价值链拆解；
- 客户决策链；
- 常见矛盾与抓手；
- 常见章节结构；
- 行业特有页型分工；
- 反复出现的能力框架、运营闭环或评价维度。

## 3. 不得直接上升的内容
- 单客户结论；
- 单项目术语；
- 未核验市场数据；
- 只出现一次的偶发写法；
- 与当前用户材料无关的泛化判断。

## 4. 升级到 `domain_profiles/` 的条件
只有满足以下条件，才可建议升级为主题知识包：
1. 跨两份以上行业材料复现；
2. 能支撑多个后续任务复用；
3. 与通用规则不冲突；
4. 不依赖单一客户或单一项目上下文；
5. 经过 `eval/reference_learning_regression.md` 回归检查。

## 5. 暂存策略
未达到升级条件的行业规律，应先标记为：
- 当前任务参考；
- 方法论候选；
- 主题知识候选。

不得直接写入全局模板。
