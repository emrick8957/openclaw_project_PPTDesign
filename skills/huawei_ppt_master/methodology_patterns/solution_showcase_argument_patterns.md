# 解决方案展示型论证模式

来源：`tmp/PPTTemplate/0517` 多行业 PPT 图片沉淀。该文件沉淀可复用论证套路，不沉淀具体客户、行业事实或未核验数据。

## 1. 战略总览—能力拆解—场景落地—生态证明

适用：对外宣讲、行业解决方案、生态合作、产品能力展示。

逻辑：

1. 战略总览：先说明趋势、机会和总体价值主张；
2. 能力拆解：把主张拆成平台、能力、机制、生态或服务模块；
3. 场景落地：按行业/业务场景说明痛点、方案、价值；
4. 生态证明：用案例、伙伴、指标、截图证明方案可落地；
5. 回到路径：以阶段计划、合作机制或规模复制作为收口。

## 2. 单场景页最小论证链

每个行业/场景页至少包含以下 3 个要素：

- 场景痛点或机会；
- 关键能力/解决方案；
- 落地价值或证明材料；
- 下一步复制路径（可选但推荐）。

缺少证明材料时，不能写成“已实现”，只能写成“可验证路径”或“待样板验证”。

## 3. 案例页角色边界

案例页只承担“证明”和“增强可信度”的作用，不替代：

- 总体方案；
- 能力架构；
- 决策建议；
- 风险与资源说明。

若用户目标是高层拍板，案例页之后必须回到“资源、责任、节奏、验收”。

## 4. 复用级别

- 可长期复用：总览—能力—场景—证明—路径的论证链。
- 局部沉淀：行业案例页、生态伙伴页、指标成果页的页面角色。
- 一次性参考：具体行业名称、客户名称、产品名称、成果数字、伙伴 Logo。
