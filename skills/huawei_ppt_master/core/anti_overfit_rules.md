# 防主题过拟合规则

## 1. 核心原则

v0.2 必须避免把 v0.1 中的昇腾/NVIDIA 样例误用到所有主题。

## 2. 禁止行为

在非 AI 算力主题中，不得无故出现：

- 昇腾；
- 英伟达；
- NVIDIA；
- CUDA；
- CANN；
- GPU/NPU；
- 模型迁移；
- 算力调度；
- 单卡性能；
- 大模型训练集群；
- 国产 AI 芯片替代。

除非用户主动要求或材料中明确包含这些内容。

## 3. 通用主题默认表达

当主题不明确时，使用通用表达：

- 业务目标；
- 当前问题；
- 建设思路；
- 总体架构；
- 核心能力；
- 实施路径；
- 风险保障；
- 决策建议。

## 4. 检查方法

生成完毕后检查：

1. 是否出现与主题无关的专属名词；
2. 是否套用了“硬件/软件/生态/模型/算力”对比结构；
3. 是否把通用客户方案写成 AI 算力方案；
4. 是否把项目复盘写成竞品对比；
5. 是否把非技术主题写得过度工程化。

出现任一问题，必须重写。

## 5. 参考材料学习过拟合控制

在持续接收 PPT、截图与行业材料时，必须额外检查：

1. 是否把单份材料中的主题术语误写成通用规则；
2. 是否把行业专属章节结构误写成所有主题默认结构；
3. 是否把单客户表达习惯误写成通用高层话术；
4. 是否因为参考材料很多，就放松了事实核验要求。

控制规则：

- 单份材料中出现的规律，默认只能视为候选规则；
- 只有跨两份及以上材料重复出现，才允许考虑进入通用层；
- 若规律只在单行业中稳定出现，应优先沉淀到主题层或方法论主题分支；
- 一旦存在主题污染风险，必须降级处理，不得强行推广。
