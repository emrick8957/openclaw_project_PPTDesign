# 主题路由规则

## 1. 默认原则

默认加载：

```text
domain_profiles/default_general.md
```

只有当用户主题明确触发关键词或语义时，才加载专属主题包。

## 2. 主题包触发条件

### ai_compute_ascend_nvidia

触发词：昇腾、Ascend、英伟达、NVIDIA、GPU、NPU、CUDA、CANN、算力、AI算力、智算、模型迁移、训练、推理、AI芯片。

### gov_public_security

触发词：公安、市局、分局、区县、政务、警情、视频解析、一网统管、社会治理、安平、公共安全。

### ai_platform

触发词：AI平台、模型平台、大模型平台、MLOps、模型管理、模型服务、推理服务、训练平台、智能体平台。

### project_operation

触发词：持续运营、运维、运营规划、运营机制、闭环、SLA、指标体系、年度运营。

### customer_solution

触发词：客户汇报、解决方案、客户方案、行业方案、售前、价值主张、交付方案。

### data_governance

触发词：数据治理、数据资产、数据中台、数据质量、数据标准、指标体系、数据目录、数据血缘。

### cybersecurity

触发词：网络安全、安全运营、SOC、主机安全、系统加固、漏洞、威胁、合规、等保、安平合规。

## 3. 多主题处理

如果同时命中多个主题包，按以下方式合并：

1. default_general 永远存在；
2. 只引入与当前主题直接相关的概念；
3. 不同主题包发生冲突时，以用户明确主题和汇报对象优先；
4. 不得把 AI 算力主题包中的专属术语带入无关主题。

## 4. 防误触发

如果用户只是说“高性能”“平台”“生态”“智能化”，不要自动触发 AI 算力主题包。

只有明确出现 AI 算力、芯片、GPU/NPU、昇腾/NVIDIA、模型训练/推理等语义，才触发。

## 5. 主题知识包升级条件

当用户持续提供某一行业或某一主题的参考材料时，不得立即写入对应 `domain_profiles/`。

只有满足以下条件，才允许升级为主题知识包：

1. 同一行业框架在两份以上材料中稳定复现；
2. 该框架不依赖单一客户、单一项目私有上下文；
3. 该框架能对后续同主题任务形成稳定复用价值；
4. 该框架与现有通用模板和反污染规则不冲突。

在达到升级条件前，应先作为临时参考或方法论候选规则处理。
