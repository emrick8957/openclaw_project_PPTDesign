# 通用视觉规则 visual_rules.md

> 定位：本文件只作为 PPT 页面视觉设计指导。  
> 作用范围：约束页面的视觉风格、色彩、版式、字体、间距、卡片、图表形态、红色锚点、页脚、安全区和负面视觉禁止项。  
> 不承担：内容事实校验、deck_spec 字段解释、生成流程控制、评分规则、证据溯源、页面文案改写等职责。

---

## 1. 总体视觉定位

### 1.1 风格关键词

- **华为高层汇报风格**
- **白底为主**
- **红黑灰白配色**
- **克制、工程化、有秩序**
- **结构清晰、层级明确、证据导向**
- **标题先形成判断，主体再证明判断**

### 1.2 页面气质

页面应呈现为正式汇报材料，而不是宣传海报或产品发布页。

推荐气质：

- 稳重；
- 理性；
- 工程化；
- 高信息密度但不拥挤；
- 有明确阅读路径；
- 有清晰主次关系。

避免气质：

- 炫技；
- 花哨；
- 娱乐化；
- 模板堆砌；
- 卡通化；
- 互联网发布会化；
- 咨询海报化。

## 2. 色彩规则

### 2.1 基础色板

主色：

- 华为红：`#C8102E`
- 深红：`#78000F`

中性色：

- 深灰：`#231815`
- 中灰：`#595757`
- 辅助灰：`#898989`
- 浅灰：`#DDDDDD`
- 极浅灰：`#F7F7F7`
- 白色：`#FFFFFF`

辅助警示色：

- 橙色：`#F8B53C`
- 深橙：`#EB5C01`

### 2.2 色彩角色

- 白色：页面主背景。
- 极浅灰：弱底块、注释区、底部轻结论区。
- 浅灰：边框、分隔线、表格线、弱连接线。
- 中灰：副标题、注释、页脚、低优先级标签。
- 深灰/黑色：正文和主要标题。
- 华为红：唯一主视觉锚点或极少量辅助强调。
- 橙色：风险、预警、阶段提醒，不与华为红并列抢焦点。

### 2.3 色彩比例

普通正文页建议：

- 白色 / 极浅灰：80%–90%
- 深灰 / 黑色：8%–15%
- 华为红：通常控制在 5% 以内
- 其他颜色：原则上不用，必须使用时面积极小

### 2.4 色彩负面禁止项

禁止：

- 大面积花哨渐变；
- 蓝紫科技渐变；
- 霓虹光效；
- 高饱和多色并列；
- 红、橙、蓝、绿同时作为重点色；
- 普通正文页使用大面积红底；
- 红色与橙色同时承担主强调；
- 使用浅粉、浅绿、浅蓝大面积柔和底色形成咨询模板感；
- 用颜色装饰页面，而不是表达层级和语义。

## 3. 页面区域与版式骨架

### 3.1 标准区域比例

- 标题区：页面上方 10%–15%
- 主体区：页面中部 75%–80%
- 页脚区：页面底部约 5%
- 左右安全区：页面左右边缘保留稳定留白
- 右侧洞察栏：如使用，宽度控制在页面宽度 25%–32%

### 3.2 标题区

标题区应承担页面第一视觉判断。

视觉要求：

- 标题左对齐为主；
- 标题 1–2 行，不宜超过 2 行；
- 主标题字重较高；
- 可使用极细红线或短红线作为页眉定锚；
- 标题下方可使用浅灰分割线；
- 标题区不得挤压主体区。

禁止：

- 使用厚重红色标题栏；
- 标题区使用大面积底色；
- 标题过长导致主体区被压缩；
- 标题、副标题、页眉全部抢焦点；
- 标题只是主题名，没有判断感；
- 标题使用多种颜色混排。

### 3.3 主体区

主体区应通过图表、卡片、流程、对比、架构、路径等视觉结构证明页面判断。

视觉要求：

- 模块对齐；
- 网格清晰；
- 主次明确；
- 阅读路径明确；
- 模块间距大于或等于卡片内边距；
- 主图区域优先于辅助说明区；
- 高密度页面也必须保留呼吸感。

禁止：

- 满屏文字；
- 满屏表格；
- 所有模块平均铺满；
- 主体贴边；
- 框中框过多；
- 左右两侧各讲一套；
- 图表、卡片、文字彼此没有视觉关系。

### 3.4 页脚区

页脚用于承载保密标识、页码、来源、版权或免责声明。

视觉要求：

- 固定在页面底部约 5% 安全区；
- 使用浅灰小字；
- 位置、字号、颜色、基线保持一致；
- 与主体区保持明显间隔；
- 不抢主图和结论。

禁止：

- 页脚位置漂移；
- 页脚字号过大；
- 页脚颜色过深；
- 页脚与底部结论条重叠；
- 页码进入主体区；
- 来源文字抢视觉。

## 4. 字体与文字层级

### 4.1 字号建议

- 主标题：32–44pt
- 副标题 / 页眉标签：12–16pt
- 模块标题：18–24pt
- 正文：14–18pt
- 图表标签：12–14pt
- 页脚 / 来源 / 注释：8–10pt

### 4.2 字重建议

- 主标题：Bold / Heavy
- 模块标题：SemiBold / Bold
- 正文：Regular / Medium
- 注释：Regular
- 页脚：Regular / Light

### 4.3 文字密度

- 普通正文页：3–5 个信息块；
- 高层总览页：4–6 个信息块；
- 单卡片正文：1–2 行为宜，最多 3 行；
- 单页 bullet 不超过 6 条；
- 右侧洞察栏不超过 3 条要点。

### 4.4 文字视觉负面禁止项

禁止：

- 正文字号低于 14pt；
- 单页长段落堆叠；
- 单卡片塞入过多文字；
- 文本行距过密；
- 中文和英文混排无层级；
- 每个词都加粗；
- 过多下划线、斜体、描边字；
- 使用口号式大字占满页面；
- 注释文字与正文同权。

## 5. 红色锚点规则

### 5.1 红色使用原则

红色是视觉锚点，不是装饰色。  
单页只允许一个主红色视觉锚点。

红色可用于：

- 标题关键词；
- 主结论；
- 关键风险；
- 核心节点；
- 最终输出；
- 阶段门槛；
- 决策动作；
- 重要路径箭头。

同一页红色只能选择一种主语义。

### 5.2 红色预算

- 主红色视觉锚点：1 个；
- 辅助红色强调：最多 2 个；
- 辅助红色必须弱于主锚点；
- 红色注意力点超过 3 个，即视为锚点分散；
- 普通正文页红色面积建议控制在 5% 以内。

### 5.3 红色层级

推荐：

- 主红色：用于唯一关键节点或关键词；
- 辅助红色：用于小图标、小圆点、细线、局部关键词；
- 弱红色：用于极浅红背景或非常细的边线，但谨慎使用。

### 5.4 红色负面禁止项

禁止：

- 同页同时出现红色标题、红色编号、红色箭头、红色卡片标题、红色底栏；
- 普通正文页红色横条成为最大视觉块；
- 普通正文页红色底座成为页面主背景；
- 多张卡片都使用红色标题栏；
- 多张卡片都使用红色粗边框；
- 大红色圆形、大红色矩形、大红色渐变承载普通说明；
- 红色编号散布全页；
- 红色图标数量过多；
- 红色箭头到处出现；
- 标题红和主图红同时抢焦点；
- 底部结论条和主图同时强红；
- 右侧洞察栏使用强红底或强红框。

### 5.5 红色纠偏方式

当红色过重时，优先采用：

- 红底改浅灰底；
- 红框改灰框；
- 红标题改黑标题 + 红关键词；
- 红箭头改灰箭头；
- 红编号改灰编号；
- 红色底栏改窄红线；
- 大红卡片改白底细红边；
- 多个红点只保留最能表达页面主判断的一个。

## 6. 卡片与模块规则

### 6.1 卡片基础样式

推荐：

- 白底或极浅灰底；
- 浅灰细边框；
- 无阴影或极弱阴影；
- 小圆角或直角；
- 统一内边距；
- 统一标题样式；
- 图标简洁线性；
- 卡片之间留白稳定。

### 6.2 卡片层级

页面中的卡片应区分：

- 主卡：最多 1 张；
- 支撑卡：2–4 张；
- 注释卡：0–2 张；
- 底部结论条：不作为主卡处理。

主卡可通过以下方式突出：

- 位置更关键；
- 字号更大；
- 边框略强；
- 留白更多；
- 与主路径连接；
- 使用唯一红色锚点。

支撑卡应保持中等权重。  
注释卡应使用灰边、无底色或极浅灰底弱化。

### 6.3 卡片视觉负面禁止项

禁止：

- 所有卡片同尺寸、同颜色、同边框、同字重；
- 多卡片页没有主次；
- 主卡过多；
- 卡片平均铺满页面；
- 卡片大圆角；
- 重阴影；
- 玻璃拟态；
- 浅粉、浅绿、浅蓝大面积卡片；
- 多层框中框；
- 卡片之间距离过近；
- 每张卡片都有红图标、红编号、红标题；
- 卡片承载流程关系但没有箭头或顺序；
- 主卡和右侧洞察栏同时成为视觉主角。

## 7. 图标与线条规则

### 7.1 图标规则

推荐：

- 线性图标；
- 单色或灰色为主；
- 少量红色点缀；
- 图标大小统一；
- 图标服务信息分类或路径节点。

禁止：

- 卡通图标；
- 拟人化图标；
- 表情符号；
- 潮流贴纸风；
- 复杂 3D 图标；
- 高饱和多色图标；
- 图标数量大于信息节点数量；
- 图标只是装饰，不承载信息角色。

### 7.2 线条规则

推荐：

- 浅灰分割线；
- 细线边框；
- 灰色连接线；
- 红色线条只用于唯一关键路径或极细定锚。

禁止：

- 粗重边框；
- 密集网格线；
- 多种线宽混用；
- 多种箭头样式混用；
- 线条贴近文字；
- 箭头只做装饰而不表达逻辑；
- 红色箭头到处使用。

## 8. 图表视觉语义规则

> 本节只约束图表的视觉表达形态，不判断内容事实。

### 8.1 通用原则

图表必须在视觉上服务页面判断。

图表应表达至少一种关系：

- 对比；
- 阶段；
- 层级；
- 流向；
- 闭环；
- 因果；
- 缺口；
- 优先级；
- 支撑关系；
- 治理动作。

禁止图表只作为装饰。

### 8.2 对比图

视觉要求：

- 左右或上下结构清晰；
- 推荐态/目标态可略强；
- 差距或转化关系要可见；
- 底部可有轻量启示区。

禁止：

- 左右两栏没有视觉差异；
- 只有并列文字，没有差异标识；
- 对比页做成大表格；
- 推荐侧过度红色强调。

### 8.3 流程图

视觉要求：

- 方向明确；
- 节点数量克制；
- 箭头样式统一；
- 起点、过程、终点清晰；
- 必要时表现回写或约束。

禁止：

- 箭头乱飞；
- 节点无主次；
- 只有名词串联；
- 多条路径无区分；
- 流程线和卡片边框纠缠。

### 8.4 闭环图

视觉要求：

- 必须形成闭合路径；
- 回写/反馈节点必须可见；
- 闭环方向必须明确；
- 可用一个红色节点标记关键回写点。

禁止：

- 把闭环画成单向流程；
- 把闭环画成分层架构；
- 环形箭头只是装饰；
- 闭环没有反馈对象；
- 多个红色节点同时抢焦点。

### 8.5 路线图

视觉要求：

- 阶段横向推进；
- 阶段之间有承接关系；
- 阶段门槛或验收产出视觉可见；
- 最终阶段可作为主红色锚点。

禁止：

- 只是并列计划；
- 阶段之间没有关系；
- 阶段卡片全部同权；
- 时间轴过度装饰；
- 红色贯穿整条时间轴导致视觉过重。

### 8.6 能力地图

视觉要求：

- 中心能力与周边场景/机制/对象有连接；
- 能力分组明确；
- 连接线少而清晰；
- 中心节点可作为唯一红色主锚点。

禁止：

- 中心一个大词，周围一圈名词；
- 连接线过多交叉；
- 周边节点同权过多；
- 能力清单伪装成能力地图；
- 红色同时用于中心节点、周边节点和箭头。

### 8.7 分层架构

视觉要求：

- 层级自下而上或自上而下清晰；
- 底座、平台、能力、应用、运营/治理关系明确；
- 层间支撑关系可见；
- 右侧说明不得压过主架构。

禁止：

- 层级堆叠但无支撑关系；
- 每层都是同一视觉权重；
- 架构图使用大量红底；
- 层内文字过密；
- 右侧洞察栏变成另一张架构图。

### 8.8 四象限

视觉要求：

- 横轴、纵轴语义明确；
- 每个象限的位置可被坐标解释；
- 目标区、风险区或优先区视觉可见。

禁止：

- 没有真实 x/y 轴；
- 只是四类概念并列；
- 象限标签过长；
- 每个象限都用强色块；
- 红色占满一个象限成为大色块。

### 8.9 表格

视觉要求：

- 线条轻；
- 字号清晰；
- 重点行/列有弱强调；
- 表格不应吞没页面结论；
- 高层页优先将大表转为面板或矩阵。

禁止：

- Excel 截图感；
- 密集网格线；
- 大表格铺满主体；
- 字号过小；
- 多行多列红色填充；
- 表格没有视觉重点。

## 9. 右侧洞察栏规则

### 9.1 视觉定位

右侧洞察栏是辅助解释区，不是第二主图。

### 9.2 推荐结构

- 1 句短结论；
- 2–3 条短要点；
- 每条不超过 2 行；
- 使用浅灰底或白底灰边；
- 图标可用灰色线性图标；
- 红色只用于极小关键词或小图标。

### 9.3 宽度规则

- 推荐宽度：页面宽度 25%–32%；
- 不得超过页面宽度 1/3；
- 与主图区保持稳定间距。

### 9.4 右侧洞察栏负面禁止项

禁止：

- 变成长备注列表；
- 超过 3 条要点；
- 每条要点过长；
- 使用强红底；
- 使用粗红框；
- 宽度过大压缩主图；
- 与主图同权；
- 内容像第二个正文区；
- 与底部结论重复；
- 图标过多导致装饰化。

---

## 10. 底部结论条视觉规则

### 10.1 视觉定位

底部结论条用于形成轻量收口，不应成为页面最大视觉块。

### 10.2 推荐样式

普通正文页推荐：

- 浅灰底；
- 白底灰边；
- 窄红线；
- 左侧小图标；
- 红色关键词；
- 黑/灰正文。

### 10.3 不同页型建议

- 普通正文页：轻量底部结论条；
- 路线图页：底部可放阶段验收口径；
- 决策页：底部可放执行提示，但拍板事项应在主体突出；
- 总结页：可稍强，但仍需克制；
- 封面/章节页：可不用底部结论条。

### 10.4 底部结论条负面禁止项

禁止：

- 厚重红色压底条；
- 底部结论条成为最大色块；
- 与页脚重叠；
- 挤压页脚安全区；
- 放过多 bullet；
- 变成第二正文区；
- 只重复标题；
- 使用过多红色关键词。

---

## 11. 图片、截图、Logo 与案例素材

### 11.1 使用原则

- 图片、截图、Logo 只能作为证明材料或辅助识别。
- 图片必须有主次。
- 案例页必须有一张主证明图或主证据区。
- 图片周围要有解释关系的文字或标注。

### 11.2 图片素材负面禁止项

禁止：

- 图片墙；
- Logo 墙；
- 截图墙；
- 多图同权排列；
- 大背景图压住文字；
- 图片只是装饰；
- 图片与标题判断无关；
- 截图无标注；
- 图片边框、阴影、圆角不统一；
- 发布会式大图 + 大口号；
- 使用卡通插画或潮流贴纸。

---

## 12. 特殊页型视觉门禁

### 12.1 封面 / 章节页

推荐：

- 大标题为主角；
- 主体极简；
- 可用短红线定锚；
- 不承载复杂论证。

禁止：

- 封面正文页化；
- 卡片过多；
- 复杂流程图；
- 背景图过重；
- 多处红色强调。

### 12.2 阅读框架页

推荐：

- 明确阅读顺序；
- 最后一格可作为管理收口；
- 模块之间有递进或聚合关系。

禁止：

- 只是目录导航；
- 所有模块同权；
- 没有主收口；
- 红色编号过多。

### 12.3 核心判断页

推荐：

- 1 张主判断卡；
- 3–4 张支撑卡；
- 主判断与支撑关系清晰；
- 底部轻量管理含义。

禁止：

- 大红主卡过重；
- 多个主卡；
- 支撑卡没有顺序；
- 右侧建议卡成为第二主角。

### 12.4 路线图页

推荐：

- 阶段推进；
- 阶段门槛；
- 验收产出；
- 最终节点作为主锚点。

禁止：

- 并列计划；
- 无承接关系；
- 无门槛；
- 无验收产出；
- 红色时间轴过重。

### 12.5 决策页

推荐：

- 拍板事项独立突出；
- 表格弱化为执行清单；
- 风险和责任视觉分层。

禁止：

- 表格吞没拍板事项；
- 决策点散落在表格中；
- 结论被底部条替代；
- 多个红色拍板项同权。

### 12.6 总结页

推荐：

- 形成最终定义、最终框架或最终判断；
- 可使用一个强收口框；
- 视觉比普通页更简洁。

禁止：

- 只复述前文；
- 只是目录回顾；
- 满页 bullet；
- 多个结论同权。

## 13. 华为风格负面禁止项总表

### 13.1 禁止互联网发布会风

表现包括：

- 大渐变背景；
- 炫光；
- 科技蓝紫；
- 霓虹线条；
- 超大口号；
- 大图压底；
- 大面积视觉冲击但缺少结构；
- 类 App 宣传页；
- 类产品发布会主视觉。

### 13.2 禁止咨询海报风

表现包括：

- 大圆角卡片；
- 柔和阴影；
- 玻璃拟态；
- 浅粉/浅绿/浅蓝大色块；
- 过多装饰图标；
- 卡片拼贴；
- 信息看似精致但逻辑主线弱；
- 每个卡片都像重点。

### 13.3 禁止卡通插画风

表现包括：

- 手绘插画；
- 拟人图标；
- 表情符号；
- 儿童绘本感；
- 潮流贴纸；
- 夸张人物造型；
- 过度可爱化。

### 13.4 禁止政务宣传风

表现包括：

- 大红底；
- 金色字；
- 飘带；
- 旗帜纹理；
- 强装饰边框；
- 庄重但缺少工程结构；
- 口号压过逻辑。

### 13.5 禁止 Excel 报表风

表现包括：

- 全页表格；
- 密集网格线；
- 字号过小；
- 无视觉重点；
- 行列信息堆叠；
- 只适合阅读，不适合汇报扫读。

### 13.6 禁止学术论文截图风

表现包括：

- 满页段落；
- 公式、表格、引用堆叠；
- 没有视觉结论；
- 仅像论文摘要重排；
- 文字密度过高。

## 14. 页面自检视觉清单

生成或设计完成后，应从视觉角度检查：

1. 是否白底为主？
2. 是否红黑灰白为主？
3. 是否只有 1 个主红色锚点？
4. 是否存在 3 个以上红色注意力点？
5. 普通正文页是否出现大红底、大红条、大红卡？
6. 标题区、主体区、页脚区比例是否稳定？
7. 页脚是否弱化且位置稳定？
8. 主体区是否有清晰阅读路径？
9. 卡片是否有主卡、支撑卡、注释卡层级？
10. 是否所有卡片同权？
11. 图表是否具备明确视觉语义？
12. 右侧洞察栏是否过宽或过长？
13. 底部结论条是否过重？
14. 是否存在图片墙、Logo 墙、截图墙？
15. 是否有互联网发布会风、咨询海报风或卡通插画风？
16. 正文字号是否不小于 14pt？
17. 模块是否贴边、拥挤或对齐不稳？
18. 是否有过度装饰性图标、线条、色块？

## 15. 视觉优化优先级

当页面不符合视觉规则时，按以下顺序优化：

1. 先收敛红色锚点；
2. 再明确主卡 / 主图；
3. 再减少文字密度；
4. 再强化阅读路径；
5. 再统一卡片样式；
6. 再弱化页脚和注释；
7. 最后处理细节装饰。

优先解决：

- 红色过重；
- 主次不清；
- 图表装饰化；
- 卡片同权；
- 页面拥挤；
- 页脚冲突。

## 16. 默认推荐样式

### 16.1 普通正文页

- 白底；
- 顶部结论型标题；
- 主体 3–5 个信息块；
- 白底/浅灰卡片；
- 浅灰细边框；
- 1 个红色主锚点；
- 底部浅灰结论条或窄红线；
- 页脚浅灰弱化。

### 16.2 核心判断页

- 左侧或上方主判断；
- 中部 3–4 个支撑判断；
- 一个轻量建议区；
- 主判断用细红边或红关键词；
- 不使用大红满底主卡。

### 16.3 流程 / 路线页

- 横向推进；
- 灰色箭头；
- 阶段节点统一；
- 最终节点红色强调；
- 底部放轻量门槛或验收口径。

### 16.4 架构 / 能力页

- 主图居中或偏左；
- 右侧洞察栏不超过 32%；
- 层级或连接关系清晰；
- 中心节点可作为唯一红色锚点；
- 避免线条过多交叉。

### 16.5 决策 / 收口页

- 拍板项或最终判断独立突出；
- 表格弱化；
- 红色用于唯一决策点；
- 底部不压重红色。

---

## 17. 一句话原则

> 华为风格 PPT 的视觉重点不是“做得炫”，而是让高层在 5 秒内看清判断、10 秒内看懂结构、30 秒内形成下一步动作。

### 18.1 0517 多行业解决方案截图的可复用边界

### 18.2 v0.3.5 生成图片反馈门禁的兼容落位

以下规则已在本文件主章节中展开，后续页面设计和图片自检必须继续遵守：

- 路线图必须体现阶段门槛、验收产出或复制条件；
- 总结页必须形成最终定义、最终框架或最终判断。

## 通用视觉规则下沉要求

通用视觉规则用于定义全局默认，不应在每页 page_design 中逐字重复并伪装成页面独有设计。

逐页 page_design 应只写：

- 本页主视觉结构；
- 本页红色锚点原因；
- 本页主图占比或区域偏离；
- 本页图表不得退化形态；
- 本页独有设计动作。

面向下游外部模型时，必须保留每页独立可读视图：

```text
page_design_view(Pn) = global_design_defaults + page_design_overrides(Pn)
```
