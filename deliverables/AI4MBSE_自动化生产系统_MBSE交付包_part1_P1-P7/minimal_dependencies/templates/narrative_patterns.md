# 通用叙事结构库

## 1. 战略规划型

封面 → 核心结论 → 外部趋势 → 内部挑战 → 目标定位 → 核心抓手 → 路线图 → 资源保障 → 风险闭环 → 决策建议

## 2. 技术方案型

封面 → 背景与目标 → 当前问题 → 总体架构 → 核心能力 → 技术路线 → 实施计划 → 运维保障 → 风险控制 → 决策事项

## 3. 竞品对比型

封面 → 总体判断 → 对比框架 → 分维度对比 → 场景适配 → 成本与风险 → 差距清单 → 机会判断 → 行动路径 → 决策建议

## 4. 客户方案型

封面 → 客户痛点 → 建设目标 → 方案总览 → 核心能力 → 场景价值 → 实施路径 → 交付保障 → 合作建议

## 5. 项目复盘型

封面 → 目标回顾 → 进展概览 → 关键成果 → 问题与原因 → 风险与影响 → 改进动作 → 下一步计划

## 6. 运营治理型

封面 → 运营目标 → 运营对象 → 指标体系 → 机制设计 → 闭环流程 → 风险处置 → 阶段计划 → 资源需求

## 7. 数据治理型

封面 → 治理目标 → 数据现状 → 关键问题 → 治理框架 → 标准体系 → 质量闭环 → 应用价值 → 推进计划

## 8. 网络安全型

封面 → 安全形势 → 资产与风险 → 能力框架 → 监测处置 → 合规要求 → 运营机制 → 建设路线 → 决策事项

## 章节规则

- 8 页以下：可以不设目录，但需要逻辑段落。
- 8~12 页：建议 3~4 个章节。
- 12 页以上：必须 4~6 个章节。
- 每个章节必须有清晰目标，不得只按页面顺序平铺。

## 9. 多行业解决方案 / 生态展示型

来源：0517 多行业解决方案截图沉淀。适用于大型企业对外宣讲、行业解决方案、生态合作、产品能力展示，不适合作为所有高层决策材料的默认结构。

推荐结构：

封面/定调 → 战略机会/行业趋势 → 总体价值主张 → 能力体系/平台底座 → 场景化落地 → 案例与生态证明 → 阶段路径/合作机制 → 愿景收口

叙事规则：

- 先用 1~2 页建立“为什么现在要做”的趋势与机会；
- 再用总览页说明“我们用什么能力体系解决问题”；
- 中段按行业/场景展开，每个场景页必须回答“痛点、方案、价值、证明”中的至少三项；
- 案例页只做证明，不替代方案论证；
- 收口页回到战略主张、合作机制或下一阶段路径。

防滥用：

- 若材料目标是内部资源决策，不应过多使用行业宣传式案例拼贴；
- 若没有真实案例/指标支撑，成果展示页必须降级为“待验证样板路径”；
- 不能把某一行业页的结构直接套到其他行业。
