# 页面版式库

## 1. executive_summary_dashboard

适用：核心结论总览、高层判断页。

结构：

- 顶部：结论型标题；
- 左侧：核心判断或关键数字；
- 中部：3~5 个模块化结论；
- 右侧：风险/建议/决策提示；
- 底部：页脚。

使用规则：适合一页讲清“现状、判断、风险、建议”。

## 2. two_column_comparison

适用：竞品对比、现状/目标对比、方案 A/B 对比。

结构：

- 左栏：对象 A；
- 右栏：对象 B；
- 中间：差异、箭头、结论；
- 底部：启示或行动建议。

## 3. four_quadrant_judgement

适用：领先/差距/风险/建议，或价值/难度/成熟度/优先级判断。

结构：2x2 矩阵 + 顶部结论 + 底部建议。

## 4. three_column_cards

适用：三类能力、三项问题、三阶段动作。

结构：三张并列卡片，每张卡片包含图标/小标题/要点/结论。

## 5. layered_architecture

适用：技术平台、数据治理、AI平台、安全运营、系统架构。

结构：底座层 → 平台层 → 能力层 → 应用层 → 运营层。

## 6. stack_architecture_with_right_insights

适用：复杂技术架构或体系架构。

结构：左/中部为分层架构图，右侧为关键洞察、风险或建设建议。

## 7. roadmap_timeline

适用：三阶段路线、年度规划、产品演进。

结构：横向时间轴 + 阶段目标 + 关键动作 + 里程碑。

## 8. trend_curve_with_strategy

适用：产业趋势、增长趋势、技术演进。

结构：左侧曲线/趋势图，右侧策略解读和应对动作。

## 9. ecosystem_map

适用：产业生态、伙伴体系、标准组织、平台生态。

结构：中心主体 + 周边角色 + 价值流/协同关系。

## 10. insight_panel_with_chart

适用：数据图、趋势图、对比图。

结构：主图表 + 右侧洞察卡片 + 底部结论。

## 11. risk_decision_matrix

适用：风险闭环、决策建议、资源申请。

结构：表格矩阵 + 重点风险标红 + 责任/时点/产出。
