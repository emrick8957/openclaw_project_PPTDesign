## 2. 共同视觉特征

1. **白底 + 红黑灰**：页面以白底为主，红色用于核心结论、关键数字、主路径和强调线。
2. **高信息密度**：多数页面信息量较大，但通过卡片、分栏、箭头、标签、底座图保持秩序。
3. **标题结论化**：标题多为判断句或趋势句，不只是名词。
4. **结构图优先**：大量使用分层架构、生态图、流程图、演进图、堆叠图、路径图。
5. **图文混排**：常见“左侧观点 + 中部图表 + 右侧洞察/案例”结构。
6. **底部页脚统一**：多保留页码、logo、保密标识或版权区域。

## 3. 可沉淀版式模式

### 3.1 高层总览仪表盘

特点：一页承载多个结论块、关键数字、结构关系和右侧洞察栏。

适用：核心结论页、总览页、战略判断页。

### 3.2 左右双栏对比

特点：左侧描述 A，右侧描述 B，中间用箭头或结论连接。

适用：竞品对比、阶段对比、现状/目标对比。

### 3.3 三栏卡片说明

特点：三个并列模块，每个模块包含小标题、图示、要点。

适用：三类能力、三阶段路径、三项举措。

### 3.4 分层架构/栈式架构

特点：底部是基础设施或平台底座，上层为能力、模型、应用、生态。

适用：技术方案、平台建设、数据治理、AI平台、安全运营。

### 3.5 趋势曲线 + 右侧策略

特点：左侧曲线说明趋势，右侧给出战略动作或能力要求。

适用：产业趋势、增长曲线、技术演进。

### 3.6 时间轴/路线图

特点：按年度、阶段或里程碑组织信息。

适用：技术演进、建设路线、产品规划、产业发展。

### 3.7 生态地图

特点：中心主体 + 周边伙伴/技术/标准/产业角色，体现关系和价值链。

适用：生态建设、合作伙伴、产业链分析。

### 3.8 风险/决策矩阵

特点：表格化呈现风险、措施、责任和时点。

适用：风险闭环、决策建议、资源申请。

## 4. 对 Skill 的增强结论
