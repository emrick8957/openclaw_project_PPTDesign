# 架构页模式

## 1. 标准分层架构

底座层、平台层、能力层、应用层、运营层。

## 2. 栈式架构 + 右侧洞察

中部为架构栈，右侧为 3~5 条建设建议。

## 3. 云边端协同架构

中心平台 + 边缘节点 + 终端/应用 + 调度与运营闭环。

## 4. 生态型架构

中心平台 + 周边生态角色 + 价值流。

## 使用规则

架构页必须说明：

- 每一层解决什么问题；
- 层与层之间如何协同；
- 哪些能力是当前重点建设对象；
- 高层需要关注的资源或风险。

## 5. 泳道流程 + 右侧洞察栏

适用：评测机制、跨角色协同、流程拆解、方法框架说明。

结构：中部 2~4 条泳道 + 右侧洞察栏。

使用规则：

- 泳道节点必须等高、等宽、等间距；
- 每个节点使用短语，不写长句；
- 箭头用细线，红色只标关键判定节点；
- 右侧洞察栏占页面宽度 25%~32%，只放 1 句结论 + 3~4 条短要点。

## 6. 价值闭环 + 右侧行动栏

适用：能力建设闭环、评测驱动闭环、治理闭环、持续改进机制。

结构：中部 4~5 节点闭环图 + 右侧行动建议。

使用规则：

- 闭环节点使用“动词 + 名词”短语；
- 闭环图只做概念锚点，详细说明放右侧行动栏；
- 右侧行动建议控制在 3~4 条；
- 底部结论区使用窄条或弱灰底，不使用厚重红色压底。
