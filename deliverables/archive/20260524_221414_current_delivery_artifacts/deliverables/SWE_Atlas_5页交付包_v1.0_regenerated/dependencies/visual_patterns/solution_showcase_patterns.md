# 解决方案展示型版式模式

来源：`tmp/PPTTemplate/0517` PPT 图片沉淀。该文件只记录可复用版式与视觉组织方式，不记录具体行业事实、客户名、项目名、产品名或原文。

## 1. 多模块方案总览页

适用：大型解决方案、行业方案、生态合作、产品能力总览。

结构：

- 顶部：结论型标题 + 一句价值主张；
- 中部：3~6 个同层级能力/价值模块；
- 底部：阶段路径、价值闭环或行动收口；
- 强调：红色只标关键判断或关键路径。

使用要点：模块必须同层级；若同时包含趋势、方案、案例、决策，应拆成多页。

## 2. 左逻辑右证明页

适用：能力证明、案例佐证、产品截图说明、问题到方案转化。

结构：

- 左侧：判断链条，通常为“问题/目标 → 抓手 → 动作”；
- 右侧：案例图、产品截图、指标、场景图；
- 底部：一句启示或下一步动作。

使用要点：右侧材料必须证明左侧判断；截图中的具体事实未核验时标注“待补材料”。

## 3. 中心辐射生态页

适用：平台生态、伙伴体系、行业协同、组织协同。

结构：

- 中心：平台/能力/组织中枢；
- 环绕：伙伴、行业、场景、能力节点；
- 连接：价值流、协同关系、数据流或运营关系；
- 侧边/底部：关键洞察与风险边界。

使用要点：必须表达真实关系，不能把普通清单包装成生态图。

## 4. 指标 + 案例展示网格页

适用：成果汇报、行业覆盖、样板项目、生态伙伴展示。

结构：

- 顶部：成果型结论标题；
- 左上或首行：2~4 个关键数字；
- 中部：案例图卡/Logo/截图网格；
- 底部：从成果回到可复制路径。

使用要点：指标必须有来源；没有数据时保留“指标位”，不得虚构数字。

## 5. 大图章节过渡页

适用：封面、章节页、愿景收口页。

结构：

- 大图或半幅图形成气氛；
- 大标题简洁有判断；
- 只保留少量副标题/日期/页脚。

使用要点：只做节奏切换，不承载复杂论证。

## 6. 防过拟合边界

- 该类版式容易偏宣传，不应直接用于内部拍板页；内部决策页需增加风险、资源、责任和验收。
- 行业图片、伙伴 Logo、成果数字不能从参考图中继承，必须由用户提供或标注待补。
- 生态图、案例图、指标网格只沉淀结构，不沉淀具体行业词。
