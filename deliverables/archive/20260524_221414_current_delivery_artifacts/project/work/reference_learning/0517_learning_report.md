## 2. 拆解结论

### 2.1 样式层

- 白底/浅灰底为主，华为红作为标题关键词、关键数字、路径箭头和节点标记。
- 页面整体信息密度高，但通过模块化卡片、细线、浅灰底块、图标和页脚保持秩序。
- 标题区通常在顶部 10%~15%，正文区承载 3~6 个信息模块，页脚约 5%。
- 大数字、图卡、案例截图、右上角标签是高频视觉锚点。

### 2.2 结构层

沉淀出 5 类高频版式：

1. 多模块方案总览页；
2. 左逻辑右证明页；
3. 中心辐射生态页；
4. 指标 + 案例展示网格页；
5. 大图章节过渡页。

### 2.3 表达层

- 标题以判断句、趋势句、能力句、价值句为主。
- 高频表达逻辑是：从旧状态走向新状态、以关键场景牵引能力演进、围绕目标打通闭环、通过样板验证后规模复制。
- 收口强调从案例/能力/成果回到可复制路径，而不是停留在宣传口号。

### 2.4 体系层

抽取出“战略总览—能力拆解—场景落地—生态证明—路径收口”的解决方案展示型论证链。

适用：对外宣讲、行业解决方案、生态合作、产品能力展示。

不适用：所有内部决策材料的默认结构。内部拍板场景必须补充资源、风险、责任、验收。
